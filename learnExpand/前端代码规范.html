前端代码规范 ​

目录命名/文件命名 ​

  1. 推荐所有目录命名采用 kebab-case（短横线连接式）
  2. 推荐所有文件命名采用 camelCase（小驼峰式命名法 —— 首字母小写）
  3. 推荐所有 vue 组件命名采用 PascalCase（大驼峰是命名法 —— 首字母大写）

VUE 单文件组件引用 ​

  1. 组件内没有内容的用自闭和标签

  2. 组件标签内有内容的，开始标签和闭合标签都要写

    vue
    <template>
  // 组件内没有内容
  <empty-component />

  // 组件标签之间有内容
  <container-component> This is content. </container-component>
</template>

  3. 引用组件时用大驼峰式命名法

    js
    <script>
import EmptyComponent from '../path';
import ContainerComponent from '../../path';

export default {
    name:'MyComponent',
    components:{
        EmptyComponent,
        ContainerComponent
	}
}
</script>

自定义事件 ​

@myEvent 会被自动转化成@myevent，事件处理函数使用 handle+动词

js
this.$emit('on-change');
vue
<my-component @on-change="handleChange" />

变量 ​

建议一般不适用 var 声明变量

js
// bad
var curTitle = '大闸蟹';

// good
let curTitle = '大闸蟹';

常量 ​

常量命名采用全大写加下划线

js
// bad
const maxCount = 10;

// good
const MAX_COUNT = 10;

引号混用 ​

  1. html 模板中使用""
  2. js 中使用''
js
// bad
let name = '大闸蟹';
let profession = 'front-end';

// good
let name = '大闸蟹';
let profession = 'front-end';
vue
// bad
<my-component data-attributes="Single quotes" />

// good
<my-component data-attributes="Double quotes" />

组件 DATA 声明为函数 ​

js
// bad
export default {
    data:{
        // ...
    }
}

// good
export default {
    data(){
        return {
            // ...
        }
    }
}

组件 PROPS 声明详细 ​

js
// bad
export default {
    props:['name','age']

    }

V-FOR 要设置 KEY ​

V-FOR 要设置 KEY 并且最好有 id 作为 key，而不是 index

vue
// bad
<ul>
    <li
      v-for="(todo,index) in todos"
      :key="index"
     >
       {{ todo.text }}
    </li>
</ul>

// good
<ul>
    <li
      v-for="(todo,index) in todos"
      :key="todo.id"
     >
       {{ todo.text }}
    </li>
</ul>

V-IF 和 V-FOR 互斥 ​

最好在另一个包裹元素上使用 v-if，或者对 v-for 的列表进行过滤

vue
// bad
<ul>
    <li
      v-for="(todo,index) in todos"
      v-if="showCurTodo"
      :key="index"
     >
       {{ todo.text }}
    </li>
</ul>

// good
<ul>
    <li
      v-for="(todo,index) in todos"
      :key="index"
     >
       <div v-if="showCurTodo">
           {{ todo.text }}
        </div>
    </li>
</ul>

多个属性时多行来写 ​

vue
// bad
<li class="item" v-for="(todo, index) in todos" @click="handleClick">
	{{ todo.text }}
</li>

// good
<li
  class="item"
  @click="handleClick"
  v-for="(todo, index) in todos"
  :key="todo.id"
>
    {{ todo.text }}
</li>

组件名为多个单词 ​

组件样式声明尽可能设置作用域 ​

由于组件开发很容易样式命名冲突

css
// bad

  // ...
}

// good
.todo-list-btn {
  // ...
}
.todo-list-btn .list-item {
  // ...
}
