uni-app 引入 Cesium 报错 ​

说明：以下是 uniapp 引入 cesuim ,在调用 cesium 时的一些问题

点击图层子节点菜单的属性后，点击表单不显示 ​

情况一控制台报错：

找到 XbsjEarthUI 目录下的 XbsjEarthUI.js 文件，删掉 t||dt&&dt._scope.off()和 t&&t._scope.on()

情况二控制台不报错：

因为在 cesuim 在 pc 端使用的组件库为 element-ui，uniapp 在打包成 app 是不支持 element-ui 组件库的，所以要改写此处表单的代码，文件大致在原 Cesium 项目目录下的 Source/components/common/form/index.vue

原代码：

html
<el-form :label-position="labelPosition" label-width="80px" :model="form">
  <el-form-item v-for="item in labelArr" :label="item.label">
    <el-input v-model="form[item.key]"></el-input>
  </el-form-item>
</el-form>

可看出此处使用了 element-ui 组件，将此处的代码进行修改，不使用 element-ui 的组件即可，可自行编写组件


打包项目在 uniapp 中引用，真机调试和 app 不显示地球 ​

问题描述： uni-app 引入 Cesium 报错，Failed to execute 'texImage2D' on 'WebGLRenderingContext'

详情说明： uni-app 引入 cesium，在 vue 页面的视图层引入 cesium 后，在 h5 端正常显示 cesium 地图，在 app 端或者手机模拟器端不显示地图，无法加载地图，报 Failed to execute 'texImage2D' on 'WebGLRenderingContext'，显示其中有图片跨域所以无法加载。

解决办法一（改方法未尝试是否有效）

版本不符合问题

  1. 使用 cesium1.83 版本测试下；
  2. 安卓版本或者手机 webview 不符合；

解决办法二（亲测有效）

直接修改打包文件夹下的 cesium.js源码，添加代码

javascript
r.crossOrigin = 'anonymous';

示例：

js
function loadImageElement(e, t, i) {
var r = new Image;
r.crossOrigin = "anonymous"; // 添加这行代码
r.onload = function () {
i.resolve(r)
}, r.onerror = function (e) {
i.reject(e)
}, t && (TrustedServers.contains(e) ? r.crossOrigin = "use-credentials" : r.crossOrigin = ""), r.src = e
}
// 差不多3026行

这里定义的是 r ，根据实际定义的变量来改

补充：

在已经尝试上述问题解决后，打包成 app 却不显示地球

思路：

可以先尝试改用自定义基座来调试看看是否显示地球，是否有报错。

解决方法：

在打包时选择传统打包。

重点说明：

在每次打包好代码到 uniapp 项目后都要进行此操作：

  1. 在 cesium.js 中添加代码
javascript
r.crossOrigin = 'anonymous';
  1. 删除 XbsjEarthUI.js 中的这两处代码
